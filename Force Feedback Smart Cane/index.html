Executive Summary

Standard white canes provide tactile ground feedback but offer no warning of obstacles at knee-to-head height. Blind users must sweep repeatedly, slowing navigation and missing overhead hazards entirely.

The Force-Feedback Smart Cane integrates stereo Time-of-Flight (ToF) sensors with a 3-zone haptic handle, providing real-time directional obstacle warnings. Users receive intuitive left-center-right vibration feedback, enabling faster, safer independent navigation.

Sensor & Haptic Architecture

Stereo VL53L1X ToF Sensors: Mounted at 15° toe-in angle on handle ends, providing continuous coverage from 20cm to 4 meters. I2C address multiplexing enables dual-sensor operation on single ESP32.

3-Zone ERM Haptic Array: Three 10mm ERM motors embedded in handle grip (Left, Center, Right). Directional vibration indicates obstacle location; intensity indicates proximity.

VL53L1X ToF ×2 ESP32-WROOM ERM 10mm ×3 AO3400 MOSFET 18650 Li-Ion

How To Build It

Complete build process from components to functional prototype. Estimated total build time: 2-3 weeks for V1 prototype.

⚠ Tools & Equipment Required

▸ Soldering Station (60W+)
▸ 3D Printer (Handle Mount)
▸ Multimeter
▸ Wire Strippers & Cutters
▸ Heat Gun / Heat Shrink
▸ ESP32 Programmer
▸ Hot Glue Gun
▸ Screwdriver Set
PHASE 1: Sensor Integration Week 1
  • Wire VL53L1X sensors to ESP32 (I2C: SDA, SCL, VIN, GND)
  • Set unique I2C addresses (0x54 for Left, 0x29 for Right)
  • Test sensor detection with Arduino VL53L1X library
  • 3D print sensor mounts with 15° toe-in angle
  • Mount sensors on handle ends with acrylic protective windows
PHASE 2: Haptic Motor Installation Week 1-2
  • Position 3× ERM motors inside handle (Left, Center, Right)
  • Solder motors to AO3400 MOSFET drain pins
  • Connect MOSFET gates to ESP32 GPIO (PWM-capable pins)
  • Add 1N4148 flyback diodes across motor terminals
  • Test vibration intensity with PWM (0-255 range)
PHASE 3: Power System Week 2
  • Wire 2× 18650 batteries in parallel (3.7V, 4000mAh total)
  • Connect TP4056 charging module to battery terminals
  • Add USB-C charging port to handle base
  • Install power switch between battery and ESP32 VIN
  • Verify voltage output (3.7V-4.2V range)
PHASE 4: Firmware Development Week 2-3
  • Flash ESP32 with custom obstacle detection firmware
  • Configure sensor sampling rate (10-20Hz)
  • Implement distance threshold logic (50cm, 100cm, 200cm)
  • Map distance zones to motor intensity (PWM 100-255)
  • Add low-battery warning vibration pattern
PHASE 5: User Testing & Calibration Week 3
  • Test with blindfolded users in controlled environment
  • Calibrate vibration intensity for comfortable feedback
  • Verify detection range accuracy (measure vs. reported)
  • Test outdoor performance (sunlight interference)
  • Iterate firmware based on user feedback
